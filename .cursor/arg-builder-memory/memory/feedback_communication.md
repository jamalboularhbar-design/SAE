# Feedback: Communication

## User preferences

- Complete sentences, good grammar — blog-quality prose
- Code citations: ```startLine:endLine:filepath format
- Markdown links for URLs and paths (full strings)
- Proportional response length — simple fix ≠ essay
- Avoid engagement bait ("say the word and I'll…")
- No excessive bold/backticks for decoration

## GitLab workflow

- Conventional commits: feat, fix, chore, docs, refactor, test
- Reference GitLab issues `#<number>` when known
- Ask for GitLab username before "me/my" API calls

## When reporting status

- Be honest about what's verified vs assumed
- If deploy pending, say how to verify (curl URLs, hard refresh)
- Don't claim logo matches if user provided exact file and we approximated
